1C Rush > Процедуры и функции

Что такое подпрограммы?

Процедуры и функции — это именованные блоки кода , которые можно вызывать многократно из разных мест программы. Они позволяют избежать дублирования кода, структурировать программу и упростить её поддержку. В 1С:Предприятие любая программа состоит из модулей, а модули — из процедур и функций. Разница между ними проста: функция возвращает значение (результат своей работы), а процедура — нет. В остальном они ведут себя одинаково: могут принимать параметры, иметь локальные переменные, быть экспортными (доступными из других модулей).

Синтаксис описания

Процедура объявляется с помощью ключевого слова Процедура, затем следует имя и список параметров в скобках. Завершается конструкция словом КонецПроцедуры. Функция отличается тем, что начинается с Функция и обязана содержать оператор Возврат. Скобки после имени обязательны, даже если параметров нет. Точку с запятой после КонецПроцедуры / КонецФункции ставить не нужно. Тело процедуры или функции может содержать любые операторы, включая вызовы других подпрограмм (вложенные объявления запрещены, только на уровне модуля).

Параметры передачи по ссылке и значению

По умолчанию параметры передаются по ссылке — изменения параметра внутри процедуры влияют на исходную переменную. Чтобы защитить исходные данные, используйте ключевое слово Знач перед параметром — тогда внутри подпрограммы будет работать копия.

По ссылке (без Знач): Процедура Изменить(Число) — изменяет исходную переменную.

По значению (со Знач): Процедура Изменить(Знач Число) — работает с копией, исходная не меняется.

Области видимости переменных

Переменные в 1С различаются по области видимости. Локальные переменные объявляются внутри процедуры или функции простым присваиванием и видны только в ней. Модульные переменные объявляются в начале модуля с ключевым словом Перем и видны всем процедурам и функциям этого модуля, но не доступны из других модулей, если не указано Экспорт. Глобальные переменные — это те же модульные, но с ключевым словом Экспорт, они доступны из любых других модулей. Важно помнить: если внутри процедуры присвоить значение переменной, которая не была объявлена локально, 1С начнёт искать её среди модульных и глобальных. Чтобы гарантированно создать локальную переменную, даже если в модуле уже есть переменная с таким же именем, используйте ключевое слово Перем прямо внутри процедуры.

Экспортные процедуры и функции

Чтобы сделать процедуру или функцию доступной из других модулей, добавьте ключевое слово Экспорт после списка параметров. Это основа для создания публичного API модуля.

Параметры по умолчанию

При описании функции или процедуры можно задать значения параметров по умолчанию. Тогда при вызове этот параметр можно опустить — он получит указанное значение. Параметры без значения по умолчанию являются обязательными.

Оператор Возврат

В функции оператор Возврат обязателен — он задаёт возвращаемое значение. Если в функции нет Возврат или он не выполнился ни в одной ветке, функция вернёт Неопределено. В процедуре Возврат можно использовать только для досрочного выхода (без указания значения после него).
